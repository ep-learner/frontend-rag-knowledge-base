# 前端 RAG 知识库 - 使用指南

## 📋 系统概述

这是一个基于 Minimax 大模型 + 本地向量存储的个人前端技术问答系统。

## 🚀 快速开始

### 1. 启动服务

**后端服务（必须先启动）**
```bash
cd server
npm run dev
```
后端服务运行在：http://localhost:3000

**前端服务**
```bash
cd client
npm run dev
```
前端服务运行在：http://localhost:5173

### 2. 访问系统

打开浏览器访问：http://localhost:5173


- 上传后立即可以在问答中使用

## 💬 问答功能使用

### 正常流程

1. **访问问答页面**
   - 点击导航栏的"问答"
   - 或从首页点击"开始问答"

2. **选择查询模式**
   - **💡 语义检索（慢）**：理解问题含义，效果好（消耗 Token）
   - **⚡ 快速检索（快）**：关键词匹配，速度快（不消耗 Token）
   - 默认使用语义检索

3. **输入问题**
   - 在输入框中输入前端技术问题
   - 例如："React Hooks 的工作原理是什么？"
   - 支持 Enter 键快速发送

4. **查看回答**
   - 系统会基于知识库自动检索相关内容
   - 使用 Minimax 大模型生成回答
   - 支持多轮对话

### 查询模式对比

| 模式 | Token 消耗 | 检索质量 | 适用场景 |
|------|-----------|---------|---------|
| 语义检索 | 消耗（Embedding API） | 高（语义理解） | 生产环境、高质量问答 |
| 快速检索 | 不消耗（本地哈希） | 中（关键词匹配） | 快速原型、离线环境 |

**注意**：
- 语义检索消耗 Embedding API Token（约 250 tokens/次）
- 快速检索不消耗 Embedding Token，但 LLM 生成回答仍消耗 Token
- 建议日常使用语义检索，确保回答质量

## 📝 日志查看功能

### 日志记录内容

每次问答都会记录以下信息：

| 字段 | 说明 |
|------|------|
| id | 日志唯一 ID |
| timestamp | 时间戳 |
| mode | 查询模式（semantic/simple） |
| userQuery | 用户问题 |
| retrievedDocs | 检索到的文档列表 |
| messages | 发送给 API 的完整上下文 |
| context | 知识库检索结果 |
| response | LLM 回答 |
| tokenUsage | Token 消耗统计 |
| processingTime | 处理时间 |
| error | 错误信息（如果有） |

### 日志查看方式

**API 接口**：
- `GET /api/chat/logs` - 查看今天的日志
- `GET /api/chat/logs?date=2026-06-22` - 查看指定日期的日志
- `GET /api/chat/logs/stats` - 查看日志统计
- `GET /api/chat/logs/:id` - 查看单条日志详情

**日志文件位置**：
- `server/logs/chat-YYYY-MM-DD.json`
- 保留 30 天的日志

### 日志用途

1. **排查模型幻觉**
   - 对比 `userQuery`、`retrievedDocs`、`response`
   - 检查检索到的文档相关性（distance 值）
   - 分析回答是否基于知识库内容

2. **Token 消耗分析**
   - 查看 `tokenUsage` 字段
   - 了解每次问答的 Token 消耗
   - 优化查询策略

3. **历史对话回放**
   - 查看 `messages` 字段
   - 了解完整的多轮对话上下文
   - 分析对话流程

## 🔧 技术架构

### 前端技术栈
- **框架**：Vite + React + TypeScript
- **样式**：Tailwind CSS
- **路由**：React Router
- **图标**：Lucide React
- **HTTP**：Axios

### 后端技术栈
- **框架**：Express + TypeScript
- **向量存储**：本地文件存储（documents.json）
- **Embedding**：Minimax Embedding API
- **LLM**：Minimax ChatCompletion API

### RAG 流程
```
用户提问 → 向量化 → 检索相似文档 → 构建上下文 → LLM生成回答
```

## ❓ 常见问题

### 1. 为什么页面显示"加载统计数据失败"？

**原因**：后端服务未启动或 API 数据格式不匹配

**解决**：
- 确保后端服务正在运行（http://localhost:3000）
- 检查后端日志是否有错误
- 刷新页面重试

### 2. 为什么文档上传失败？

**原因**：
- 文件格式不支持（只支持 .md）
- 后端服务未启动
- 文件内容为空

**解决**：
- 确保文件是 Markdown 格式
- 检查后端服务状态
- 查看浏览器控制台错误信息

### 3. 问答功能为什么没有响应？

**原因**：
- 后端服务未启动
- Minimax API 配置错误
- 知识库为空

**解决**：
- 检查后端服务状态
- 检查 `.env` 文件中的 API 配置
- 确保已执行 `npm run upload` 上传文档

### 4. 如何添加新的知识文档？

**方法1：手动上传**
- 在前端"文档管理"页面选择文件上传

**方法2：批量上传**
- 将文档放入 `knowledge-docs/` 目录
- 执行 `npm run upload` 命令
