# 前端安全与性能平衡

## 核心知识点

### 安全性能权衡

在前端开发中，安全和性能需要平衡：

1. **XSS 防护**：转义用户输入，但可能影响渲染性能
2. **CSP 策略**：限制资源加载，但可能阻止必要的脚本
3. **HTTPS**：加密传输，但增加握手开销
4. **输入验证**：在客户端验证，但可能影响用户体验

### 安全最佳实践

| 安全措施 | 性能影响 | 优化方案 |
|----------|----------|----------|
| 内容安全策略 | 阻止未授权脚本 | 使用严格的白名单 |
| 输入验证 | 增加客户端计算 | 使用高效正则表达式 |
| HTTPS | 增加握手时间 | 使用 HTTP/2 和 TLS 1.3 |
| 加密存储 | 增加计算开销 | 使用 Web Cryptography API |

### 性能安全监控

监控安全事件和性能指标，及时发现问题：
- 安全事件日志
- 性能监控告警
- 定期安全审计

const sanitize = (input) => {
  const div = document.createElement('div');

  return div.innerHTML;

const validateEmail = (email) => {
  const re = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return re.test(email);
};

async function encryptData(data) {
  const encoder = new TextEncoder();
  const dataBuffer = encoder.encode(data);
  
  const key = await crypto.subtle.generateKey(
    { name: 'AES-GCM', length: 256 },
    true,
    ['encrypt', 'decrypt']
  );
  
  const iv = crypto.getRandomValues(new Uint8Array(12));
  const encrypted = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv: iv },
    key,
    dataBuffer
  );
  
  return { encrypted, iv, key };
}

## 常见问题

### Q: CSP 策略太严格导致脚本无法加载怎么办？

A: 使用 CSP 报告模式（report-only）先测试，再逐步调整策略。

### Q: HTTPS 对性能的影响有多大？

A: 首次连接有握手开销，后续连接可复用，HTTP/2 可缓解。
