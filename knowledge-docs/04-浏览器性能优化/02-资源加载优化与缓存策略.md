# 资源加载优化与缓存策略

## 核心知识点

### 资源加载优化

资源加载优化的目标是减少页面加载时间：

1. **代码分割**：按需加载代码
2. **懒加载**：延迟加载图片和组件
3. **预加载**：提前加载关键资源
4. **压缩**：压缩 CSS、JS、图片

### 缓存策略

HTTP 缓存分为两种：

1. **强缓存**：通过 Cache-Control 或 Expires 实现
2. **协商缓存**：通过 Last-Modified/ETag 实现

### Service Worker

Service Worker 可以实现离线缓存和推送通知：
- 拦截网络请求
- 缓存资源
- 实现离线访问

```javascript
const sw = '/service-worker.js';

if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register(sw)
    .then(registration => {
      console.log('Service Worker registered');
    })
    .catch(err => {
      console.error('Service Worker registration failed:', err);
    });
}
```

```javascript
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open('v1').then((cache) => {
      return cache.addAll([
        '/',
        '/index.html',
        '/styles.css',
        '/app.js'
      ]);
    })
  );
});

self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request).then((response) => {
      return response || fetch(event.request);
    })
  );
});
```

## 常见问题

### Q: 强缓存和协商缓存怎么选？

A: 静态资源用强缓存，动态内容用协商缓存。

### Q: 什么时候使用 preload 和 prefetch？

A: preload 用于当前页面的关键资源，prefetch 用于后续页面的资源。
