# 浏览器渲染原理与性能优化

## 核心知识点

### 浏览器渲染流程

浏览器渲染页面的过程分为以下步骤：

1. **HTML 解析**：解析 HTML 构建 DOM 树
2. **CSS 解析**：解析 CSS 构建 CSSOM 树
3. **布局（Layout）**：结合 DOM 和 CSSOM 计算布局
4. **绘制（Paint）**：绘制像素到屏幕
5. **合成（Composite）**：合并图层并显示

### 渲染优化策略

| 优化策略 | 说明 |
|----------|------|
| 减少重排 | 使用 transform 替代 top/left |
| 减少重绘 | 使用 will-change 提示浏览器 |
| 分层渲染 | 将频繁变化的元素独立为图层 |
| 懒加载 | 延迟加载非关键资源 |

### 关键渲染路径

关键渲染路径是指从 HTML 到首次渲染的过程，优化目标是最小化以下时间：
- DOMContentLoaded
- Load
- First Contentful Paint

## 代码示例

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Optimized Page</title>
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap" rel="stylesheet">
  <style>
    .layer {
      will-change: transform, opacity;
      transform: translateZ(0);
    }
    
    .animate {
      transform: translateX(100px);
      transition: transform 0.3s ease;
    }
  </style>
</head>
<body>
  <div class="layer">
    <h1>Optimized Content</h1>
    <p class="animate">This element will animate smoothly</p>
  </div>
  
  <script defer>
    document.addEventListener('DOMContentLoaded', () => {
      console.log('DOM ready');
    });
  </script>
</body>
</html>
```

## 常见问题

### Q: 重排和重绘的区别是什么？

A: 重排影响布局，重绘只影响像素绘制，重排成本更高。

### Q: 如何判断元素是否触发了重排？

A: 使用 Chrome DevTools 的 Performance 面板，勾选 Paint flashing。
