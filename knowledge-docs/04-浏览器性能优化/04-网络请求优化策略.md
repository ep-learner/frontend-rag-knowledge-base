# 网络请求优化策略

## 核心知识点

### HTTP/2 特性

HTTP/2 相比 HTTP/1.1 的优势：

1. **多路复用**：单个连接处理多个请求
2. **头部压缩**：减少请求头大小
3. **服务器推送**：主动推送资源
4. **优先级**：设置请求优先级

### 请求优化

网络请求优化的目标是减少请求数量和大小：

1. **合并请求**：使用 HTTP/2 或合并静态资源
2. **压缩传输**：使用 gzip 或 brotli
3. **减少重定向**：避免不必要的跳转
4. **使用 CDN**：加速静态资源访问

### 连接优化

1. **TCP 连接复用**：减少握手次数
2. **HTTP/3**：使用 QUIC 协议
3. **DNS 预解析**：提前解析域名

```javascript
const controller = new AbortController();

const fetchData = async () => {
  try {
    const response = await fetch('/api/data', {
      signal: controller.signal,
      headers: {
        'Accept': 'application/json',
        'Content-Type': 'application/json'
      },
      method: 'GET',
      cache: 'default'
    });
    
    const data = await response.json();
    return data;
  } catch (error) {
    if (error.name === 'AbortError') {
      console.log('Request aborted');
    } else {
      throw error;
    }
  }
};

const request = fetchData();

setTimeout(() => {
  controller.abort();
}, 5000);

const cache = new Map();

async function getData(url) {
  if (cache.has(url)) {
    return cache.get(url);
  }
  
  const response = await fetch(url);
  const data = await response.json();
  
  cache.set(url, data);
  return data;
}
```

## 常见问题

### Q: HTTP/2 和 HTTP/1.1 的主要区别是什么？

A: HTTP/2 支持多路复用、头部压缩等特性，性能更好。

### Q: 如何取消一个 fetch 请求？

A: 使用 AbortController 的 abort 方法。
