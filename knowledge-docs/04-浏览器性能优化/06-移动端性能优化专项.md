# 移动端性能优化专项

## 核心知识点

### 移动端性能特点

移动端有以下性能限制：

1. **CPU/GPU 资源有限**：移动设备性能远低于桌面
2. **内存限制**：移动端内存较小，容易 OOM
3. **网络不稳定**：移动网络延迟高、带宽波动大
4. **电池续航**：性能优化直接影响续航

### 移动端优化策略

| 优化策略 | 说明 |
|----------|------|
| 响应式设计 | 适配不同屏幕尺寸 |
| 触摸优化 | 优化触摸事件和手势 |
| 内存管理 | 及时释放不再使用的资源 |
| 省电模式 | 降低动画和刷新频率 |

### PWA 优化

渐进式 Web 应用的性能优化：
- 使用 Service Worker 缓存
- 实现离线访问
- 优化首屏加载速度

```javascript
const handleTouch = (event) => {
  event.preventDefault();
  
  const touch = event.touches[0];
  const { clientX, clientY } = touch;
  
  console.log(`Touch at: ${clientX}, ${clientY}`);
};

document.addEventListener('touchstart', handleTouch, { passive: true });

let lastTap = 0;

const handleDoubleTap = (event) => {
  const currentTime = new Date().getTime();
  const tapLength = currentTime - lastTap;
  
  if (tapLength < 300 && tapLength > 0) {
    console.log('Double tap');
  }
  
  lastTap = currentTime;
};

document.addEventListener('touchend', handleDoubleTap);

function throttleAnimation() {
  let lastTime = 0;
  
  return function(callback) {
    const currentTime = new Date().getTime();
    
    if (currentTime - lastTime > 16) {
      callback();
      lastTime = currentTime;
    }
  };
}

const throttledRender = throttleAnimation();

function render() {
  throttledRender(() => {
    updateUI();
    requestAnimationFrame(render);
  });
}

const supportsPassive = (() => {
  let supports = false;
  try {
    const opts = Object.defineProperty({}, 'passive', {
      get() {
        supports = true;
      }
    });
    window.addEventListener('test', null, opts);
  } catch (e) {}
  return supports;
})();

document.addEventListener('touchmove', handleTouchMove, {
  passive: supportsPassive ? true : false
});
```

## 常见问题

### Q: 移动端为什么要使用 passive 事件监听器？

A: 启用 passive 后，浏览器可以立即滚动，提升触摸性能。

### Q: 如何检测设备性能并降级？

A: 使用 Performance API 和 Feature Detection。
